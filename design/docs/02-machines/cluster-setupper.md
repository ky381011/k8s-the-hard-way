# Cluster Setupper

## 概要
Cluster Setupperは、Kubernetesクラスターを構成するコントロールプレーンとワーカーノードを初期化・構築するための管理マシンです。すべてのクラスターセットアップ処理をこのマシンから一元管理し、Client Setupperが提供するAnsibleプレイブックにより自動化されます。

## 用途
- Kubernetesクラスターを構成するノード（コントロールプレーン、ワーカーノード）の初期化・構築
- クラスターコンポーネント（etcd、API Server、Kubelet等）のインストール・設定
- クラスター内ノード間の通信設定とネットワーク構成
- Client Setuperから送信されるAnsibleプレイブックの実行

### 責務
| 責務 | 説明 |
|------|------|
| **ノード初期化** | ターゲットノードのOS設定、パッケージ管理システムの準備 |
| **証明書管理** | クラスター内通信用のCA、サーバー証明書の生成と配布 |
| **コンポーネントインストール** | etcd、API Server、kubeletなどのインストール |
| **クラスター構成** | ノード間の接続設定、ネットワークプラグインの配置 |
| **検証・確認** | セットアップ完了後のクラスター動作確認 |

### 機能要件
1. **ターゲットノードとの通信**
   - SSH公開鍵認証によるセキュアな接続
   - 複数ノードへの並行アクセス

2. **クラスターコンポーネント管理**
   - etcdの初期化と起動
   - API Serverの設定と起動
   - コントローラーマネージャー、スケジューラーの設定
   - kubeletの配置と設定

3. **証明書・秘密鍵管理**
   - CA証明書の生成
   - コンポーネント間通信用の証明書生成
   - 秘密鍵の安全な配布

4. **ネットワーク構成**
   - ノード間通信の確保
   - CNI（Container Network Interface）プラグインの設定

5. **クラスター検証**
   - コンポーネントの起動確認
   - ノード登録の確認
   - 基本的な動作テスト

### 非機能要件
| 要件 | 説明 |
|------|------|
| **セキュリティ** | 公開鍵認証、証明書によるセキュアな通信 |
| **スケーラビリティ** | 複数ノードの同時セットアップ対応 |
| **再現性** | Ansibleによる自動化で同一環境を再現 |
| **監視・ログ** | セットアップ処理の詳細ログ記録 |
| **べき等性** | 何度実行しても同じ結果になる |

## 要件

<details>
<summary>詳細</summary>

- 当マシンのみでクラスターのセットアップが可能
  - クラスターの操作を行うことができるツールがそろっている
- ターゲットノードへの公開鍵認証による接続
- クラスター内の全ノードと通信可能なネットワーク環境

</details>

## セットアップフロー
1. Client Setupperから初期パラメータを受け取る
2. Ansible Inventoryの生成
3. 事前チェック（ターゲットノードへの接続確認）
4. ノードの初期設定（OS設定、パッケージ更新）
5. 必要なツールのインストール
6. 証明書の生成と配布
7. etcdクラスターの構成
8. Control Planeコンポーネントの配置
9. Worker Nodeの登録
10. クラスター検証と確認
